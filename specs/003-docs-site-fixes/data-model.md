# Phase 1 Data Model: Docs Site Fixes & Plugin Verification

This feature has no runtime data model. The "entities" are the documentation artifacts and tracking
records the work operates on. They are modeled here so tasks and verification map cleanly to them.

## Entity: Navigation Topic

A primary sidebar entry defined in `astro.config.mjs` via `starlight-sidebar-topics`.

| Field       | Description                                            |
| ----------- | ------------------------------------------------------ |
| `label`     | Display name (e.g. "Reference")                        |
| `icon`      | Sidebar icon (e.g. `information`, `seti:brackets`)     |
| `link`      | Target URL at section root (e.g. `/reference/`)        |
| `directory` | Autogenerated content directory backing the topic      |
| `resolves`  | Derived: does `link` return HTTP 200 on the live site? |

**Instances**: Getting Started (`/guide/`), Reference (`/reference/`), Architecture (`/architecture/`),
API Reference (`/api/`), Changelog (`/changelog/`).

**Validation rule**: every Topic where `link` is a section root MUST have a backing Section Landing
Page so `resolves = true`.

## Entity: Section Landing Page

The `index.md` at a section root.

| Field         | Description                                        |
| ------------- | -------------------------------------------------- |
| `path`        | e.g. `docs/src/content/docs/reference/index.md`    |
| `title`       | Frontmatter title                                  |
| `description` | Frontmatter description                            |
| `tags`        | Optional frontmatter tags (feeds starlight-tags)   |
| `badge`       | Optional frontmatter page badge                    |
| `body`        | Intro prose + links into the section's child pages |

**Instances to create**: `reference/index.md`, `architecture/index.md`, `api/index.md`.
**Existing reference instance**: `guide/index.md` (template).

**Validation rules**: links in `body` must resolve (link validator); page must build; must coexist
with auto-generated siblings (api/).

## Entity: Plugin Verification Record

One per installed Starlight plugin. Tracked in `contracts/plugin-verification-matrix.md`.

| Field        | Description                                                            |
| ------------ | ---------------------------------------------------------------------- |
| `plugin`     | Package name (e.g. `starlight-github-alerts`)                          |
| `feature`    | What it provides                                                       |
| `activation` | Content/config needed to demonstrate it (if any)                       |
| `evidence`   | Observed proof on the live site (rendered control / output / behavior) |
| `status`     | `Working` \| `Fixed` \| `Removed`                                      |
| `reason`     | Required when `status = Removed`                                       |

**Validation rules**: `status` MUST NOT be Unknown; `Removed` MUST carry a `reason`; intended outcome
is all records Working/Fixed.

## Entity: Deferred Verification Task

A manual check carried over from spec 002.

| Field      | Description                                                       |
| ---------- | ----------------------------------------------------------------- |
| `id`       | Spec-002 task id (T029, T030, T035, T043, T045, T051, T053, T055) |
| `scenario` | What to exercise                                                  |
| `outcome`  | `pass` \| `fail`                                                  |
| `followup` | Recorded item when `outcome = fail`                               |

**Validation rule**: every task MUST end with `pass` or a recorded `followup`.

## Relationships

```text
Navigation Topic ──backed by──> Section Landing Page (1:1 for section-root topics)
Section Landing Page ──may host──> activation content for ──> Plugin Verification Record(s)
Plugin Verification Record ──verified via──> live browser + astro build
Deferred Verification Task ──verified via──> live browser + deploy/build checks
```
