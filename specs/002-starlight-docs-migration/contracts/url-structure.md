# Contract: URL Structure

**Feature**: 002-starlight-docs-migration
**Type**: URL preservation contract (SC-001)

This contract defines the URL paths that MUST continue to resolve after the VitePress → Starlight migration. Any URL that existed in the VitePress site must return HTTP 200 (or a 301 redirect to the canonical Starlight URL).

---

## Trailing Slash Convention

Starlight generates URLs with trailing slashes by default. The deploy workflow must configure GitHub Pages or the static hosting layer to handle this consistently (either always trailing slash or always no trailing slash — not mixed).

**Chosen convention**: trailing slash (Starlight default). CI should NOT strip trailing slashes.

---

## Base Path

If deployed to a GitHub Pages subpath (e.g., `https://rocketSurgeonsGuild.github.io/Indago/`), the Astro `base` config must be set to `/Indago` and all internal links must be relative (Starlight handles this automatically when `base` is configured).

---

## llms.txt Contract

The `llms.txt` file at `/llms.txt` MUST list all non-draft documentation pages in this format (generated by `starlight-llms-txt`):

```
# Indago Documentation

> Compile-time assembly scanning for .NET

## Documentation

- [Home](/): Landing page and feature overview
- [Installation](/guide/installation/): How to install Indago
- [Quickstart](/guide/quickstart/): First-time usage guide
- [AOT Publishing](/guide/aot-publishing/): Publishing with Native AOT
...
```

Each line follows the format: `- [Title](URL): Brief description`
