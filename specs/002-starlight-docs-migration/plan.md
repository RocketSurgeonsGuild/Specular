# Implementation Plan: Migrate Documentation Site to Starlight

**Branch**: `002-starlight-docs-migration` | **Date**: 2026-06-26 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/002-starlight-docs-migration/spec.md`

---

## Summary

Replace the VitePress documentation site with an Astro Starlight site. The migration recovers existing content from git history (`0f4311b`), installs and configures 14 Starlight plugins, integrates Typesense-powered search, generates API reference from compiled .NET DLLs via xmldocmd, and updates the CI deployment workflow. No generator, public API, or test changes are required — this is a documentation infrastructure feature.

---

## Technical Context

**Platform**: Astro 5.x with Starlight 0.35+
**Node.js**: ≥ 20.0.0 (managed by mise)
**Package structure**: Separate `docs/package.json` for Astro project; root `package.json` retains formatter tooling only
**Search**: Typesense Cloud (managed); `starlight-docsearch-typesense` plugin
**API doc generator**: xmldocmd (XmlDocMarkdown) — dotnet global tool, pure Markdown output
**Content recovery**: Historical docs available at git commit `0f4311b` (guide/, reference/, architecture/ sections)
**Testing**: Build-time validation (link validator, Astro build) + CI smoke tests; no unit tests required for docs tooling
**Target Platform**: GitHub Pages (static site, trailing slash convention)

## Phase 1: Foundation & Content Recovery

**Goal**: Working Starlight site with migrated content, no plugins yet.

### T001 — Remove VitePress, Initialize Astro + Starlight

- Remove `docs/.vitepress/` directory entirely
- Remove `docs/api-examples.md` and `docs/markdown-examples.md`
- Initialize Astro project in `docs/` with Starlight template
- Configure `astro.config.mjs` with basic site metadata (title, description, GitHub social link)
- Verify `npm run dev` starts in `docs/`

### T002 — Recover Historical Docs Content

- Recover content from git at `0f4311b` (guide/, reference/, architecture/ + index.md)
- Move files into `docs/src/content/docs/` directory structure
- Convert home page frontmatter from VitePress format to Starlight splash template (see data-model.md)
- Remove VitePress-specific syntax; verify all pages render in dev server

### T003 — Configure Basic Sidebar and Navigation

- Configure sidebar in `astro.config.mjs` with auto-generated groups for guide/, reference/, architecture/
- Set up social links (GitHub repo URL), site title, favicon
- Verify sidebar shows all migrated pages correctly

### T004 — Update Mise Tasks and CI Workflow

- Update `mise.toml` tasks: `docs`, `docs:build`, `docs:api`
- Update `.github/workflows/deploy-docs.yml` for Astro build + dotnet build prerequisite
- Add Typesense indexing step (credentials from CI secrets)

**Milestone validation**: `mise run docs` starts server; all content pages render; `mise run docs:build` completes.

## Phase 2: Plugin Integration

**Goal**: All 14 plugins installed and configured.

### T005 — Quality & Validation Plugins

- **starlight-links-validator**: fail build on broken links
- **starlight-auto-drafts**: exclude `draft: true` pages from production
- **starlight-github-alerts**: render GitHub alert syntax; convert any VitePress callout syntax found

### T006 — Navigation & Structure Plugins

- **starlight-sidebar-topics**: 6 topic groups (Getting Started, Guides, Reference, Architecture, API, Changelog)
- **starlight-tags**: enable tagging; verify `/tags/` index page
- **starlight-changelogs**: wire CHANGELOG.md into changelog/ content area

### T007 — Search: Typesense

- **starlight-docsearch-typesense**: replace default search
- Create Typesense Cloud collection; document env vars in `.env.example`
- Configure CI secrets; add indexing step to deploy-docs.yml

### T008 — Content Enhancement Plugins

- **starlight-heading-badges**: badge frontmatter support
- **starlight-image-zoom**: click-to-zoom for images/diagrams
- **starlight-scroll-to-top**: scroll-to-top on long pages
- **starlight-page-actions**: "Edit on GitHub" + "View Source" actions
- **starlight-plugin-icons**: icon support in MDX

### T009 — LLM Discoverability

- **starlight-llms-txt**: generate `/llms.txt` listing all non-draft pages

**Milestone validation**: All 14 plugins render correctly in both light and dark mode (SC-007).
